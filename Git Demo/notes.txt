1. Clone - cloning repos on our local machine
        - git clone <-some link->

2. status - displaythe state of the code
        - git status

3. add - adds new or changed files in your working directory to Git staging area
        - git add <-file name->
        - git add . = to add all

4. commit - record of change
        - git commit -m "some message"

5. push - upload local repo content to remote repo
        - git push origin main


Create new repo
1. init - used to create a new git repo
        - git init
        - git remote add origin <-link->
        - git remove -v (to verify remote)
        - git branch (to check all branches)
        - git push origin main

Branch command
1. git branch (to check all branches)
2. git branch -M <-name-> (to rename branch)
3. git checkout <-branch name-> (to navigate)
4. git checkout -b <-new branch name-> (to create new brach)
5. git branch -d <-branch name-> (to delete branch)

Merging branches 
1. git diff <-branch name-> (to compare commits, branches, files & more)
2. get merge <-branch name-> (to merge 2 branches)
or
2. Create a Pull request

Fixing Mistakes
Case 1 : staged changes
        - git reset <- file name ->
        - git reset
Case 2 : commited changes (for one commit)
        - git reset HEAD~1
Case 3 : commited changes (for many commits)
        - git reset <- commit hash ->
        - git reset --hard <- commit hash ->
